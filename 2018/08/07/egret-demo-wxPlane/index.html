游戏玩法


这个打飞机的 demo 是用 egret 仿写微信小游戏的示例的，所以只可以拿来当做是练习额~~



玩法很简单，就是用户拖动飞机，击败敌机。用户飞机自动发射子弹，敌机也可以发射子弹，用户击败 30 / 60 架敌机，分别可以一次发射 2 / 3 颗子弹。用户飞机被敌机子弹击中或被敌机撞到，游戏结束。

目录介绍


目录大致和上次的 racingCar demo 差不多，都有 common, control, events, utils, view 等目录。
这次还有几个类，是根据微信小游戏的实例仿写过来的，所以这次类的创建，是按照他的示例来创建的：
1. npc/Enemy.ts 敌机类、
2. player/Player.ts 用户机类、
3. player/Bullet.ts 子弹类、
4. 对象池 common/Pool.ts 类，控制当前页面对象（子弹和敌机）的数量，避免使用 js 原有的垃圾处理机制，而是通过对象池来复用已经创建的对象，算是一个性能优化。
5. 还有 Databus.ts 类，管控游戏状态，包含分数、子弹数组、敌机数组、是否游戏结束等。



主要代码讲解

npc/Enemy.ts 敌机类
x 轴随机出现敌机，并向下移动、发射子弹、被击中时，播放 movieClip 爆炸动效、碰撞检测函数

player/Player.ts 用户机类
创建用户飞机、用户 level （根据 level 发射不同的子弹数量）、发射子弹、碰撞检测

player/Bullet.ts 子弹类
这个类比较简单了，就只有初始化其位置和速度，然后更新子弹的位置。

对象池 common/Pool.ts 类
使用对象池，创建敌机和子弹类，可以回收对象，不用重复创建多个对象，优化性能。

Databus.ts 类
游戏的数据状态，管理游戏数据。reset 重置游戏数据、回收子弹和敌机

分别创建了游戏中需要用到的类后，就可以在 GameScene.ts 页面写游戏的玩法逻辑了。

GemeScene.ts
游戏时场景页，监听用户交互（用户滑动，飞机也跟着移动），监听 ENTER_FRAME 事件，每帧进行子弹和飞机的移动，并检测是否发生碰撞，执行相应操作。

开发时遇到问题：

  1. 在用户 touchMove 时，碰到到敌机或敌机子弹时，游戏结束，弹出结束面板，此时如果用户手指还在屏幕上，就没有执行了 touchEnd 事件，所以没有移除 touchMove 事件，用户还可以滑动飞机（如果 over 面板设置了 touchEnabled = true），所以要在弹出 over 面板时，移除 touchMove 事件。
