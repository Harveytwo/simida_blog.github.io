关系链数据使用指南


今天来总结一下写微信小游戏的排行榜。
微信小游戏，可以借助于微信的开放数据域，丰富了游戏的社交玩法，比如：好友之间的游戏排名、群组里面成员排名等。

一个微信用户的关系链数据包括两部分：

  1. 该用户好友的用户数据，wx.getFriendCloudStorage() 获取当前用户也玩该小游戏的好友的用户数据
  2. 该用户所在的某个群的群成员的用户数据，wx.getGroupCloudStorage() 获取当前用户在某个群中也玩该小游戏的成员的用户数据

返回的数据，如下图：

更多详细的介绍，看官方文档

总结：
其实，就是通过调用 wx.setUserCloudStorage() 把用户数据托管于微信的平台，然后需要的时候，向微信发起请求，拉取相关数据（这里数据是没有排序的，需要自己处理）。
但是，当小游戏启动开放数据域，小游戏环境会对主域和开放数据域应用一些限制。

限制：

  1. sharedCanvas 只能被绘制到上屏 canvas 上。
  2. 上屏 canvas 不能调用 toDataURL，其 context 不能调用 getImageData。
  3. sharedCanvas 不能调用 toDataURL 和 getContext。
  4. 不能将上屏 canvas 和 sharedCanvas 以任意形式绘制到其他 canvas 上，包括 drawImage、createPattern、texImage2D、texSubImage2D。
  5. sharedCanvas 的宽高只能在主域设置

这里有几个概念解释一下：

  1. 开放数据域：是一个封闭、独立的 JavaScript 作用域。要让代码运行在开放数据域，需要在 game.json 中添加配置项 openDataContext 指定开放数据域的代码目录。
  2. 主域：除 src/myOpenDataContext（这个是开放数据域代码目录） 以外是 主域的代码目录。
  3. 上屏 canvas ：在整个小游戏代码中首次调用 wx.createCanvas() 创建的是上屏 Canvas，之后调用则创建的是离屏 Canvas。如果你的项目中使用了官方提供的 Adapter 即 weapp-adapter.js，那么你此时创建的会是一个离屏 Canvas。因为在 weapp-adapter.js 已经调用了一次 wx.createCanvas()，并把返回的 canvas 作为全局变量暴露出来。
  4. sharedCanvas ：是主域和开放数据域都可以访问的一个离屏画布。在开放数据域调用 wx.getSharedCanvas() 将返回 sharedCanvas。而在主域中可以通过开放数据域实例访问 sharedCanvas，通过 drawImage() 方法可以将 sharedCanvas 绘制到上屏画布。

注意：

  1. 开放数据域不能向主域发送消息。
  2. 主域可以向开放数据域发送消息
  3. 所以开放数据域只能监听主域发送过来的消息，然后根据类型，触发不同的行为。
