游戏玩法

该笔记记录做一个简单的游戏项目，赛车游戏。

赛车游戏，

  1. loading 记载页，加载进来
  2. 游戏开始页，点击开始按钮，进入游戏页面
  3. 倒计时3、2、1，车灯闪烁，用户左右滑动，小车随着左右移动，躲避路上其他车辆
  4. 根据配置，设置开了 10000km，算是完成了游戏，显示有成功的游戏结束页面，如撞到其他车辆，则显示失败的游戏结束页面，可点击再玩一次，返回首页。

这里策划了一个简单的赛测游戏，熟悉 egret 的使用。接下来就开始写代码了。

目录介绍


创建游戏项目（不是 eui 额），根据之前学到的 egret 一些知识，新建一些文件和目录。

该项目自定义 loading ui，使用了场景切换的特殊效果（这两个在之前的文章中有写到）。这里有几个文件需要介绍一下:

  1. common/Common.ts 这是放一些公共的变量或者函数的，方便开发统一管理
  2. utils/MoveUtil.ts utils 目录是放一些工具类，赛车游戏，需要用到移动车辆，所以创建一个移动的工具类
  3. Data.ts 这是该游戏项目的一些数据
  4. Game.ts 一些该游戏独有的变量或函数，和 Common.ts 相似，不过这个只是针对该项目

上面这些分类，是个人喜欢，你也可以自己根据你的喜好，创建那些类。

common/Common.ts 公共的变量或者函数

utils/MoveUtil.ts utils
这个类比较长，这里就不粘贴了，代码地址会在后面附上。

Data.ts 游戏相关数据定义

Game.ts 这里我只是写了一个点击按钮效果：按钮点击放大动画，移开取消

主要代码讲解


创建了基础类后，就开始写该项目的主要逻辑了。

  1. 创建所有的游戏场景 view/*.ts，实现各个场景间的切换，保证可以跳转全部场景
  2. 该项目比较简单，就 3 个场景，主要操作是在赛车场景 SceneGame.ts
  3. 游戏开始场景 SceneBegin.ts 比较简单，就需要监听开始按钮事件即可
  4. 游戏结束场景 SceneEnd.ts 也较简单，监听再来一次事件，且更加用户是否完成任务，显示成功或失败的图标

SceneBegin.ts 和 SceneEnd.ts 这里就不重复了，之前的场景切换写的比较详细了。

主要讲讲 SceneGame.ts
梳理一下这页面的主要逻辑：

  1. 倒计时 3、2、1
  2. 车辆移动
  3. 用户左右滑动
  4. 计算米数
  5. 碰撞检测
  6. 游戏结束判断，成功或失败

先创建背景，无缝循环移动的背景，两个背景拼接起来，背景移动，用户车辆不移动，视觉上感觉用户车辆再移动。

创建用户车辆，

倒计时，使用 tween 动画就可轻松实现了

我们现在创建了背景、用户车辆、倒计时，接下来，就是游戏开始了，还记得我们每个 场景都有一个 start 方法控制事件监听吧~~这里监听用户左右滑动的操作和监听每一帧触发事件 ENTER_FRAME ，开始记录用户的米数。

ENTER_FRAME 事件，这个每一帧都触发的事件，在这里监听用户车辆的碰撞、背景移动、随机产生其他车辆、移动和回收清除它们、判断是否游戏结束（成功或失败）

还有一个类，其他车辆的工厂类 CarFactory.ts，可以批量创建和清楚回收车辆。
